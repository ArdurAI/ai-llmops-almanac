The market is bifurcating: **visual no-code platforms** (Dify, Flowise, n8n, Gumloop) for rapid prototyping vs. **code-first frameworks** (LangChain, LangGraph, CrewAI, Mastra, Pydantic AI) for production scale. The migration arc is real—teams start in no-code, hit complexity ceilings, and rewrite in code. The winners are tools that bridge both worlds or own one end of the spectrum completely.

The most accessible entry point for LLMOps. These tools compete on visual graph fidelity, integration breadth, and self-hosting viability.

| **Flowise** | 54.9K | Apache-2.0 | LangChain-native; drag-and-drop; Workday-backed | SQLite concurrency data loss; no native loops; enterprise paywall |
| **n8n** | 188K | Fair-code | 400+ integrations; AI agent nodes; EU-based | Fair-code not OSI-approved; queue mode complexity; frequent releases |
| **Gumloop** | — | Proprietary | AI-native; 130+ nodes; multi-model routing; ~$70M funded | Cloud-only; pricing opaque; no self-hosting below Enterprise |
| **Zapier** | — | Proprietary | 7,000+ app integrations; 3M+ users; 69% Fortune 1000 | Task-based pricing surprises; no HIPAA; cloud-only |

Flowise's Workday acquisition (Aug 2025) adds enterprise credibility but raises questions about open-source roadmap independence.

## Key Findings

### 1. Evaluation-first replaced "ship and pray"
Braintrust reports 30%+ accuracy improvements for customers within weeks of adoption. The industry has shifted from "build workflows and hope they work" to "evaluate every prompt change before deployment." This is the single biggest cultural shift in LLMOps in 2025–2026.

### 2. No-code → code-first migration arc is real and painful
Teams prototype in Dify/Flowise/n8n, then migrate to LangChain/LangGraph/CrewAI when hitting scale ceilings. The migration is painful because visual workflows don't translate to code—nodes, edges, and state management have no 1:1 mapping. Tools that bridge this gap (Dify's API, Flowise's programmatic access) will win.

### 3. Chinese open-source platforms are gaining traction
BISHENG, FastGPT, and RAGflow collectively represent the strongest non-Western LLMOps ecosystem. BISHENG explicitly positions itself as a Chinese alternative to Dify. Fortune 500 deployments in China are real. These tools are under-reported in Western benchmarks.

### 4. Capital is concentrating in late-stage rounds
82.5% of disclosed funding goes to Series C+ rounds; median $230M. The seed/A stage is barren. This means new entrants need either (a) massive differentiation or (b) open-source community traction to survive. Pure proprietary plays without network effects are dying.

### 5. Observability is becoming infrastructure
Langfuse + ClickHouse, Braintrust's $800M valuation, and the rise of "tracing as a first-class citizen" in every framework (LangSmith, Helicone, Opik) show that observability is no longer a nice-to-have. It's the control plane for LLM applications.

### 6. AutoGen is in maintenance mode—Microsoft is consolidating
Microsoft Research's AutoGen (55K+ stars) is officially in maintenance mode as of Oct 2025. The successor is the unified Microsoft Agent Framework, combining Semantic Kernel + AutoGen. This is a warning for teams building on AutoGen directly—migrate to the new framework or to LangGraph/CrewAI.

### 7. Pezzo is showing warning signs
No major commits since June 2025. Python SDK incompatible with modern LLM features. This is a classic open-source risk—small team, no sustainable funding model, community moves on. If you're evaluating Pezzo, budget for a migration path.

### 8. The "visual vs. code" debate is false
The real divide is "stateful vs. stateless." Stateful tools (LangGraph, Temporal, Semantic Kernel) handle long-running, retryable, durable workflows. Stateless tools (most no-code builders, simple API chains) fail silently when a node errors mid-execution. Production LLM workflows are inherently stateful—this is why Temporal is on this roster despite being a general workflow engine.
